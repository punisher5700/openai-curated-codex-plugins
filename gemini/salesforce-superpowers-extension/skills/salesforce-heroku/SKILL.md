# Salesforce Heroku

Design Salesforce and Heroku solutions with clear operational boundaries between CRM workflows and custom application runtime concerns.

## Focus

- Heroku app architecture and worker model
- Heroku Postgres and data boundary design
- Salesforce integration and event-driven coordination
- Platform API, background processing, and async orchestration
- Deployment, scaling, and operability concerns

## Rules

- Keep CRM transaction logic separate from app runtime concerns.
- Use Heroku when custom runtime flexibility is actually needed.
- Be explicit about source-of-truth and sync boundaries.

## Required Output

1. Runtime architecture
2. Data ownership model
3. Integration model
4. Scaling and ops considerations
5. Deployment guidance
