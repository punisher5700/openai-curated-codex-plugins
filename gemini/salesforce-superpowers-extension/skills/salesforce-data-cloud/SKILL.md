# Salesforce Data Cloud

Design Data Cloud solutions around identity, governance, and activation outcomes rather than just ingestion.

## Focus

- Data ingestion and source onboarding
- Identity resolution and profile unification
- Harmonization, mappings, and calculated insights
- Segmentation and activation
- Data governance, retention, and ownership boundaries
- Integration with core CRM, Marketing Cloud, and external data platforms

## Rules

- Be explicit about identity resolution assumptions.
- Separate raw ingestion from unified profile outcomes.
- Define activation targets, freshness expectations, and governance controls.
- Call out when Data Cloud is the wrong tool for operational CRM transactions.

## Required Output

1. Source systems
2. Identity model
3. Harmonization model
4. Activation strategy
5. Governance and retention model
6. Integration boundaries
