# Salesforce Industries

Design industry-cloud solutions that respect packaged data models, process orchestration, and sector-specific workflow complexity.

## Focus

- Industry data model extensions
- Guided processes and orchestration
- Sector-specific product, service, or case lifecycles
- Packaged capability boundaries versus custom extension
- Integration and data-governance implications

## Rules

- Reuse packaged capabilities before introducing heavy custom overlays.
- Be explicit about what stays in the industry package model versus custom objects and logic.
- Watch upgradeability and packaged-process complexity.

## Required Output

1. Industry capability mapping
2. Data model strategy
3. Process orchestration model
4. Extension boundaries
5. Upgrade and governance considerations
