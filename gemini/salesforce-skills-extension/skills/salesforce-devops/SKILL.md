# Salesforce DevOps

Design DevOps workflows that fit Salesforce metadata behavior and team release reality.

## Focus

- Repo and package strategy
- Branching and environment promotion
- Automated validation and test execution
- Team ownership and release governance
- Metadata conflict management and deployment risk reduction

## Rules

- Optimize for repeatable delivery, not heroic manual deploys.
- Align source boundaries with ownership and release cadence.
- Call out metadata types that challenge modular delivery.

## Required Output

1. Repo and package model
2. Environment promotion path
3. Validation strategy
4. Release governance model
5. Operational tradeoffs
