# Salesforce Experience Cloud

Design external experiences that are secure, understandable, and supportable.

## Focus

- Site structure and external user journeys
- Guest, partner, and customer access models
- Sharing, visibility, and content exposure
- Authentication and identity patterns
- Self-service workflows and case/community interactions
- Performance and operational support considerations

## Rules

- Treat guest access as high risk by default.
- Be explicit about user type, role model, and record visibility.
- Separate community UX concerns from core CRM data ownership.

## Required Output

1. User model
2. Access model
3. Site and page model
4. Data visibility rules
5. Security and operational considerations
