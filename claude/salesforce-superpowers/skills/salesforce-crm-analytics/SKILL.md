# Salesforce CRM Analytics

Design analytics that are trusted, refreshable, and aligned with business decisions rather than just dashboard assembly.

## Focus

- Dataset and semantic design
- Recipes, transforms, joins, and refresh strategy
- Dashboard, lens, and KPI design
- Security inheritance and analytics visibility
- Data latency, metric definition, and stakeholder trust
- Integration with core CRM and external analytical sources

## Rules

- Define metric meaning before building dashboards.
- Design refresh timing and source dependencies explicitly.
- Keep dataset logic explainable and testable.
- Call out where analytics visibility diverges from CRM visibility.

## Required Output

1. Source and dataset model
2. Metric and KPI definitions
3. Refresh and transformation strategy
4. Dashboard and consumption model
5. Security and governance notes
