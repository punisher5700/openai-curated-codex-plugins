# Salesforce Revenue Cloud

Design revenue operations for lifecycle correctness, pricing clarity, and manageable downstream finance integration.

## Focus

- Subscription and contract lifecycle
- Pricing, amendments, renewals, and expansions
- Quote-to-cash and order-to-revenue boundaries
- Revenue operations process design
- Integration with ERP, billing, and fulfillment systems
- Governance around entitlements, renewal timing, and commercial state changes

## Rules

- Be explicit about commercial state transitions.
- Separate quoting concerns from billing and finance system ownership.
- Treat amendments and renewals as first-class design flows.
- Clarify contract truth, order truth, and billing truth when they differ.

## Required Output

1. Lifecycle model
2. Commercial state transitions
3. System ownership boundaries
4. Integration architecture
5. Operational and financial control points
