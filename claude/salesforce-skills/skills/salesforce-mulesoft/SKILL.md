# Salesforce MuleSoft

Design MuleSoft solutions around clean API boundaries, operability, and ownership.

## Focus

- API-led connectivity
- System, process, and experience API decomposition
- Transformation and orchestration boundaries
- Policy enforcement, auth, and API governance
- Resilience, retries, observability, and error channels
- Salesforce as producer, consumer, or orchestrated participant

## Rules

- Keep APIs purpose-specific and ownership-aligned.
- Avoid collapsing all business logic into one orchestration layer.
- Make retries and idempotency explicit.
- Treat operational visibility as a design requirement.

## Required Output

1. API layer design
2. Ownership boundaries
3. Auth and policy model
4. Orchestration and transformation design
5. Resilience and observability plan
